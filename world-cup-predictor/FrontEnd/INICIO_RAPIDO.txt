🚀 CÓMO EMPEZAR
═══════════════

PASO 1: Configurar URL de API
   Editar: js/config.js
   Cambiar: API_BASE_URL: 'http://localhost:8080/api'
   (por la URL de tu Spring Boot)

PASO 2: Iniciar Servidor
   Opción A - Python (recomendado):
   $ cd FrontEnd
   $ python -m http.server 8000

   Opción B - PHP:
   $ php -S localhost:8000

   Opción C - Node.js:
   $ npx http-server

   Opción D - Script automático:
   $ chmod +x setup.sh
   $ ./setup.sh

PASO 3: Acceder en Navegador
   URL: http://localhost:8000

🔌 INTEGRACIÓN CON SPRING BOOT
════════════════════════════════

Este frontend está diseñado para trabajar con un backend Spring Boot.

Endpoints Requeridos:
   POST   /api/auth/register     ← Registrar usuario
   POST   /api/auth/login        ← Login
   POST   /api/auth/refresh      ← Refrescar token
   GET    /api/teams/list        ← Listar equipos
   POST   /api/matches/predict   ← Realizar predicción
   GET    /api/matches/history   ← Historial de predicciones

Ver: INTEGRACION_SPRING_BOOT.md para detalles completos

📊 FLUJO DE LA APLICACIÓN
═════════════════════════

1. Usuario abre index.html
   └─> Verifica si tiene JWT válido
       ├─ SÍ → Redirige a dashboard.html
       └─ NO → Muestra landing page

2. Usuario selecciona "Registrarse"
   └─> Va a register.html
       ├─ Completa formulario
       ├─> Envía a POST /api/auth/register
       └─> Redirige a login.html

3. Usuario selecciona "Iniciar Sesión"
   └─> Va a login.html
       ├─ Completa formulario
       ├─> Envía a POST /api/auth/login
       ├─> Recibe JWT tokens
       ├─> Guarda en localStorage
       └─> Redirige a dashboard.html

4. En Dashboard
   ├─> Carga lista de equipos
   ├─> Muestra formulario de predicción
   ├─> Usuario selecciona 2 equipos
   ├─> Presiona "Predecir"
   ├─> Envía a POST /api/matches/predict
   ├─> Recibe resultado con probabilidades
   └─> Muestra resultados


🧪 PRUEBAS MANUALES
═══════════════════

1. Registro
   • Ir a http://localhost:8000/html/register.html
   • Completar formulario (sin espacios en usuario)
   • Debe redirigir a login después de registro exitoso

2. Login
   • Ir a http://localhost:8000/html/login.html
   • Usar credenciales registradas
   • Debe redirigir a dashboard

3. Dashboard
   • Debe mostrar tu nombre en navbar
   • Selecciona dos equipos diferentes
   • Presiona "Predecir"
   • Debe mostrar resultado (si el backend está listo)

4. Logout
   • En navbar, presiona "Cerrar Sesión"
   • Debe redirigir a login

⚠️ PUNTOS IMPORTANTES
══════════════════════

1. URL de API
   ⚠️ Cambiar en js/config.js antes de usar
   La URL por defecto apunta a localhost:8080

2. CORS en Backend
   ⚠️ Debe permitir llamadas desde http://localhost:8000
   Ver INTEGRACION_SPRING_BOOT.md

3. JWT Secret
   ⚠️ Debe ser la misma en frontend y backend
   Frontend solo consume, backend genera

4. HTTPS en Producción
   ⚠️ Cambiar URLs a https:// en producción
   Usar tokens en httpOnly cookies si es posible
